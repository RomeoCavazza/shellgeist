================================================================================
shellgeist — Technical Specification & Logic Dictionary
================================================================================

--------------------------------------------------------------------------------
1. Core Architecture
--------------------------------------------------------------------------------
Architecture Pattern
  Decentralized: UI (Lua) decoupled from Agent Loop (Python) via Unix Domain Sockets.

Frontend (nvim/lua/shellgeist/)
  sidebar.lua: Chat/audit UI using nui.nvim.
  rpc.lua: JSON-lines socket client implementation.
  conflict.lua: Inline diff/conflict resolution panel.
  health.lua: Check for daemon and model availability.

Backend (backend/shellgeist/)
  Agentic core managing state, tool lifecycle, and LLM communication.
  runtime/server.py: Unix socket server (RequestHandler.__call__).
  cli.py: Main entry point for the `sgd` daemon.

Communication
  JSON-lines protocol over `/tmp/shellgeist.sock` (default).

--------------------------------------------------------------------------------
2. Agent Reasoning (backend/shellgeist/agent/)
--------------------------------------------------------------------------------
loop.py (Agent Loop)
  orchestrates user goal -> final response.
  _MAX_HISTORY_OBS_CHARS (2800): Cap for tool results in history.
  _MAX_HISTORY_ASSISTANT_CHARS (4000): Cap for agent responses.
  _PROTECTED_DOC_BASENAMES: [README.md, LICENSE, docs/] - require verification.

orchestrator.py (Cognitive Logic)
  classify_model_turn: Tool call vs Response vs Salvage.
  salvage_slope_to_tool_calls: Recovers XML tags from raw LLM output.
  decide_no_tool_action: Recovery planning when no tools are selected.

--------------------------------------------------------------------------------
3. Tool Library (backend/shellgeist/tools/)
--------------------------------------------------------------------------------
executor.py
  Executes batches, handles errors, and formats observations for the loop.

fs.py (Filesystem)
  read_file, write_file, list_files, find_by_name. Strict schema enforcement.

shell.py (Execution)
  run_shell, exec_shell_session. Sandbox-aware.

edit.py (Refinement)
  edit_file: Natural language based diff generation.

--------------------------------------------------------------------------------
4. Research & Logic Flows
--------------------------------------------------------------------------------
Intent Classification
  FAMILY_MODEL (Probabilistic) vs FAMILY_HEURISTIC (Deterministic).

Self-Repair Cycle
  ObserveResult -> IsSuccess -> (if No) RepairOnce -> Re-execute.

--------------------------------------------------------------------------------
5. Environment & Lifecycle
--------------------------------------------------------------------------------
flake.nix
  Reproducible dev environment with Python 3.11, Lua 5.1, and project dependencies.

Usage Commands
  shellgeist / sgd     Start the backend daemon.
  :SGSidebar           Toggle UI.
  :SGAgent             Start task.
  :SGMode auto/review  Switch autonomy level.
